# Agentabook Foundation

Agentabook is building the operating layer for agent-native organizations.

This repository is the public whitepaper and coordination project for that thesis.

This repository is a public whitepaper project for builders, researchers, and agent developers who want to help define:

- how agent-native organizations run,
- how know-how becomes a first-class operating asset,
- how internal collaboration and external coordination should work,
- how open transaction networks emerge after strong collaboration layers exist.

## Why This Matters

Most AI systems still frame agents as features inside products.

We think the more important transition is organizational.
As agents become persistent workers, reviewers, coordinators, and decision-support units, the missing layer is no longer only better models.
The missing layer is a native operating structure for organizations that increasingly run through agents.

## Working Thesis

Agentabook is not only an agent marketplace.

Agentabook is a platform for agent-native organizational operating units.

Inside one organization, the core problem is collaboration.
Across organizations, the core problem is coordination and transaction.

That leads to a three-part system model:

- GitHub layer: internal execution and collaboration for working agents
- DataHub layer: internal know-how, governance, context, and management for R&D and management agents
- TransactionHub layer: inter-organization coordination, settlement, attribution, and value exchange

## Why This Repo Exists

This repo is intended to do three things:

1. make the category legible,
2. make the system model discussable,
3. attract aligned contributors to help build it.

This repository is intentionally public.
It is meant to become a serious working paper and coordination surface, not a static manifesto.

## Current Narrative Arc

The current whitepaper makes six claims:

1. agents are becoming operating units, not only software features,
2. organizations need internal collaboration and governance layers before open markets,
3. the core entity is an agent-native organizational operating unit,
4. GitHub and DataHub patterns solve internal organizational problems,
5. TransactionHub patterns solve inter-organization coordination and exchange,
6. the long-term market is built on know-how accumulation, not shallow task listings.

## How To Participate

If you agree with the direction and want to help, start in one of these ways:

1. read the whitepaper and open an issue on a missing assumption,
2. propose clearer object models or interface boundaries,
3. contribute a real organizational use case,
4. help refine the bilingual language and terminology,
5. challenge the roadmap if you think the phase order is wrong.

See [CONTRIBUTING.md](CONTRIBUTING.md), [OPEN_QUESTIONS.md](OPEN_QUESTIONS.md), and [DISCUSSIONS.md](DISCUSSIONS.md).

## What We Want From Collaborators

We are looking for people who want to contribute at one of three levels:

- thesis: challenge or sharpen the argument
- spec: help define objects, interfaces, and protocols
- use cases: bring real organizational workflows, agent patterns, and market needs

We are especially interested in contributors who can connect the thesis to concrete operating reality.
Abstract agreement is less useful than real workflows, real constraints, and real disagreements.

## Non-Goals For This Repo

This is not a business plan.
This is not a token paper.
This is not a final product specification.
This is not a pitch deck in markdown form.

It is a public working foundation for the Agentabook thesis.
