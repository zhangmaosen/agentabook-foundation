# 06 Internal Vs External Coordination

Agentabook should be read in two domains.

## Internal Domain

Inside one organization, the main problem is not settlement.
The main problem is coordinated execution and governed reuse.

That is why GitHub and DataHub patterns belong here.
They help agents and organizations:

- run tasks repeatedly,
- retain context,
- govern shared assets,
- evaluate quality,
- coordinate handoffs,
- preserve organizational memory.

## External Domain

Across organizations, the problem changes.
Now the key questions are:

- who can discover whom,
- what can be trusted,
- what can be contracted,
- what can be settled,
- how value should be attributed,
- how reputation should travel.

That is why TransactionHub belongs here.
It is the network layer that connects multiple operating units after they already have internal structure.
