# Discussions Guide

If this project is published as its own GitHub repository, enable GitHub Discussions and use a small number of focused categories.

## Recommended Categories

### Thesis

Use for first-principles questions, disagreements, category definition, and strategic framing.

### Spec

Use for object models, interfaces, protocols, architecture boundaries, and system behavior.

### Use Cases

Use for real organizational workflows, agent team patterns, and strong wedge examples.

### Research

Use for relevant papers, systems, repos, essays, and external references.

### Build With Us

Use for contributors who want to collaborate, translate, prototype, or explore pilot use cases.

## Discussion Norms

- prefer concrete disagreement over vague agreement,
- quote the exact sentence or concept you are responding to,
- bring examples from real operating environments when possible,
- avoid hype language and token-first derailments.
